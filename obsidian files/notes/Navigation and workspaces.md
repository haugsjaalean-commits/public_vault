


## The interface of Obsidian

The interface of Obsidian is exceedingly simple. It's so simple, in fact, that it feels condescending to even take the time to explain it, but I already started explaining it, so I might as well finish. Feel free to skip/skim this section if you already feel comfortable with the interface. 



![[Navigation and workspaces-1.png]]


In the above image, I have outlined the different sections of Obsidian's interface with different colors. 
- In green, on the left hand side, we see the toolbar, which contains multiple useful options. 
- In pink, on either side of the screen, we see the side bars, which can be collapsed and expanded. The sidebars contain native functionalities of Obsidian as well as functionalities added by [[community plugins]]. (Some of the key functionalities include: file explorer, search bar, graph view, calendar view, backlink view, etc.) The different options can either be split into separate tabs or grouped into different options contained in one tab. It is important to note that any and all options in the side bars can be moved to the main window portion of the screen, and tabs can also be moved to the side bars.
- In red, at the center of the screen, are the tabs, which allow us to view our notes, images, [[canvases]] and [[bases]]. This is where the main action will be happening. There is an option in the top right corner of each note which allows you to switch between editing mode and reading mode (otherwise called viewing mode). 



## A note on workspaces

### What is a workspace?

A workspace is simply the configuration and positioning of the different UI elements and tabs on the screen (which are detailed above under "The interface of Obsidian"). I have certain preferences when it comes to my personal configuration of these elements, which I explain below. 




### Saving workspaces


Workspaces can be saved using the option in the toolbar on the left hand side of the screen. However, it is important to note that any changes to your workspace will be automatically saved for your next session, so this option is not strictly necessary. It is only useful to use this if you have multiple different desired workspaces between which you would like to be able to toggle quickly. (The only time I use this is when I have a bunch of tabs open referring to a given subject and I want to save them so I can dive into a different subject and be able to return without having to open ten tabs all over again.)

## My workspace
My workspace is nothing special; in fact, it barely differs from the default layout. 
### The side bars
#### The right hand side bar

##### The top of the right hand side bar
Obsidian is all about seeing the connections between notes, so I always like to have the backlinks of a file in view. This is why I dedicate the entire top portion of my right hand side bar to the backlinks view. This way, I always have an idea of where my note exists positionally in my vault.

![[Navigation and workspaces-2.png]]



 In addition to having this view, you can enable a view of backlinks at the bottom of every file in the settings menu: 
 
![[Navigation and workspaces-3.png]]

##### The bottom of the right hand side bar

I dedicate the bottom of the right hand sidebar to a plethora of different views and options. These include: **tag view** (which is enhanced by the [[community plugins|community plugin]] ***Tag Wrangler***), **calendar view**, **recent files**, **properties view**, **outgoing links**, **outline** and my **local graph** (the local graph can be opened with the command menu (`Ctrl + P`)).

#### The left hand side bar

The configuration of my left hand side bar is completely default. It consists of one window which can be toggled between the **file explorer** and the **search menu** (and the **bookmarked files** which I never ever use thanks to the ***Homepage*** [[community plugins|community plugin]]).


### The tool bar

The options available in the tool bar are decided by which addons you have enabled, and they can also be hidden at will by right clicking on a blank space under the tool bar:
